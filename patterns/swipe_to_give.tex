\newcommand{\desc}{Ein Datenobjekt auf einem Sender-Gerät soll auch auf einem Empfänger-Gerät verfügbar sein.}

\newcommand{\solution}{Der Benutzer hat ein Quellgerät in der Hand oder vor sich stehen. Durch eine Wischbewegung (Swipe) von unten nach oben, werden Daten auf ein zweites Gerät, das Zielgerät, übertragen. Die Übertragung hängt mit der Ausrichtung der Geräte zusammen.}

\newcommand{\useraction}{Der Benutzer hält Gerät A (Sende-Gerät) in der Hand oder hat das Gerät vor sich stehen (z.B. in Form eines Tabletops).\\ 
Der Benutzer führt einen Swipe mit einem Finger vom unteren zum oberen Bildschirmrand aus. Die Bewegung erfolgt in Richtung des Empfänger-Gerätes, welches die Daten empfängt. Der Swipe hat eine Länge, sowie eine Richtung. Dies ist zwingend erforderlich, um eine Datenübertragung anzuregen. 
}

\newcommand{\reaction}{Ist der Datensatz erfolgreich versendet worden, so reagiert Gerät A mit einer visuellen Nachricht an den Benutzer. Er bekommt eine kleine Nachricht angezeigt, dass die Übertragung erfolgreich verlief. Bereits während des Swipes soll der Benutzer verdeutlicht bekommen, dass die ausgewählten Dateien übertragen werden. Hierfür verblassen die ausgewählten Dateien nach oben hin.\\
Gerät B empfängt den Datensatz und zeigt diesen auf dem Bildschirm an. Außerdem wird ein kurzer Vibrationsalarm ausgelöst. Auch hier könnten die Dateien vom oberen Rand hineingleiten.}

In diesem Fall wird der Swipe mit einem Finger ausgeführt und findet in einer eigens dafür gestalteten App statt. Um die Interaktion, den Swipe, durchführen zu können, wird eine Verbindung zu einem Gerät benötigt. }
