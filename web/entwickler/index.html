ENTWICKLER

Das Ziel einer Multiscreen-Interaktion ist die intuitive, Gesten-basierte Interaktion zweier Geräte zwecks Verbindungsaufbau, Dateiübertragung, Bildschirmerweiterung oder Verbindungsaufbau. Das Kapitel "Technisches" in den Pattern-Beschreibungen enthält Pattern-spezifische Hinweise zur Implementierung der jeweiligen Gestenerkennung. Diese Hinweise müssen im Kontext einer Multiscreen-Applikation umgesetzt werden, dessen Aufbau auf dieser Seite modellhaft nach [Schneider 2015] beschrieben wird.

Lebenszyklus einer Multiscreen-Applikation

Jede Instanz einer Multiscreen-Applikation kann sich in verschiedenen Zuständen befinden, wobei es aus Sicht von Multiscreen-Interaktionen vier relevante Zustandsänderungen (Lifecycle-Events) gibt, die durch Gesten ausgelöst werden können:

  • Connect: Verbindung zu einem anderen Gerät wird hergestellt, im Folgezustand sind beide Geräte verbunden
  • Select: Ein Datenobjekt wird zur Übertragung ausgewählt. Es kann sich dabei um eine Datei, einen Befehl oder einen Datenstrom handeln (bspw. zur Bildschirmerweiterung) handeln
  • Transfer: Das ausgewählte Datenobjekt wird übertragen
  • Disconnect: Die Verbindung zwischen zwei Geräten wird getrennt
